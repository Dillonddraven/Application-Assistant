# Application Answers

### How has your education and project work prepared you for this role?
My coursework targeted what entry-level analysts do day to day. In an incident-response course I built a playbook that started from a business impact analysis, identified sales as the client's essential function, and sequenced infrastructure-restoration steps so sales-supporting systems came back online first. I also designed a tiered cloud backup schedule that placed frequently accessed data in fast-retrieval storage and infrequently accessed data in cheaper archival tiers, balancing recovery time against ongoing cost. Combined with my Dillard's internship work on SAST ingestion and vulnerability reporting, my Graylog and OpenSearch centralized logging lab with rsyslog and NXLog forwarding and alerts, Wireshark-based traffic analysis from the Accurate Interpretations role, and CMMC v2 and NIST SP 800-171 documentation work, I am ready to monitor events, triage findings, and escalate when needed.

### Describe your experience with security logs and SIEM.
I configured log forwarding from Linux and Windows endpoints into a centralized logging environment, validated event ingestion, reviewed authentication activity, and troubleshot cases where expected events were not appearing. I also built a Python script to classify log entries into clearer categories, which improved visibility into system activity and made it easier to identify security-relevant patterns. That work covered the full monitoring flow, from generating events and forwarding logs to classifying activity and using the results to support investigation.

### Describe your experience with vulnerability assessment and remediation tracking.
I built reporting that turned vulnerability data into remediation guidance. During my information security internship, I helped build a SAST reporting workflow that ingested findings from GitLab repositories and converted raw scan output into clean Excel reports with severity breakdowns, team and repository views, and visual summaries. The value was making vulnerabilities understandable so stakeholders could prioritize remediation based on severity, ownership, and visibility instead of sorting through raw scanner output.
