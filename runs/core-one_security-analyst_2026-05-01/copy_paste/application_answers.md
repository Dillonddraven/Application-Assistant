# Application Answers

### How has your education and project work prepared you for this role?
My coursework and projects targeted the day-to-day work of an analyst. I built a centralized logging lab with Graylog, rsyslog, and NXLog, validated event ingestion, and set alerts for Linux sudo/auth activity and Windows failed logons. I used Wireshark for traffic analysis during an onsite internship, and I turned code-scanner output into stakeholder-ready remediation reports during my Dillard's internship. I also completed CMMC v2 and NIST SP 800-171 documentation work and designed a tiered cloud backup schedule that balanced recovery time with ongoing cost. Together, these prepared me to monitor events, turn findings into clear remediation priorities, and communicate risk to stakeholders.

### Describe your experience with vulnerability assessment or remediation tracking.
I helped build a SAST reporting workflow that ingested findings from company GitLab repositories and converted raw scan output into clean Excel reports with severity breakdowns, team and repository views, and visual summaries. The value was making the data understandable so stakeholders could see active vulnerabilities across repos, prioritize remediation by severity and ownership, and avoid sorting through raw scanner output.

### Describe your experience working with security logs.
I configured log forwarding from Linux and Windows endpoints into a centralized logging environment, validated event ingestion, and reviewed authentication activity. I built a Python script to classify log entries into clearer categories, which improved visibility into system activity and made it easier to identify security-relevant patterns. That work covered generating events, forwarding logs, validating collection, classifying activity, and using the results to support investigation.
